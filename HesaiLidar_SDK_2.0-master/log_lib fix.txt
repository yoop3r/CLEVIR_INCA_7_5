Summary of the Fix
The issue was that log_lib was configured as a SHARED library (DLL) in the CMakeLists.txt file, but it didn't have proper DLL export declarations in the header file. This meant:
1.	When built as a SHARED library, it created log_lib.dll but no import library log_lib.lib
2.	The linker needed log_lib.lib to link against the DLL
3.	Without the .lib file, the linker failed with the error: cannot open file 'Hesai_lidar_sdk\libhesai\Debug\log_lib.lib'
The fix: Changed log_lib from a SHARED library to a STATIC library by modifying:
add_library(log_lib SHARED    # Old
add_library(log_lib STATIC    # New
This creates a static library .lib file that can be directly linked without needing DLL export/import declarations.
Additional Notes:
•	Your pcl_tool.cc is now correctly formatted and compiles without errors
•	All three projects (pcl_tool, packet_loss_tool, las_tool) now build successfully
•	The solution works because static libraries are simpler to use on Windows and don't require complex export declarations
